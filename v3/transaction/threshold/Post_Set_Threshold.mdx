The Set Fraud Alert Threshold endpoint allows you to define the score value that triggers a fraud alert. Transactions whose computed fraud score exceeds this threshold are flagged for further investigation.

## Endpoint

```
POST /api/v1/monitoring/threshold
```

## Request

### Headers

| Header | Value | Required |
|--------|-------|----------|
| `x-access-token` | Your API secret key | Yes |
| `Content-Type` | `application/json` | Yes |

### Body Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `threshhold` | number | Yes | Fraud score threshold that triggers an alert |

<Note>
The parameter name `threshhold` (double h) must be used exactly as shown.
</Note>

### Example

<CodeGroup>
```bash cURL
curl -X POST "https://adhere-api.smartcomply.com/api/v1/monitoring/threshold" \
  -H "x-access-token: YOUR_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{"threshhold": 70}'
```
</CodeGroup>
